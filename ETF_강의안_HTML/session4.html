GitHub에 내 프로젝트 공유하기

⏱️ 30분
🎯 이 회차를 마치면: 내가 만든 프로젝트가 GitHub 웹사이트에 올라가서, 주소만 알려주면 누구나 볼 수 있어요! 전체 강의 완성!

Step 1. GitHub가 뭔지 알아보기 (3분)

📌 GitHub(깃허브)는 "코드의 인스타그램"이에요!
• 인스타그램에 사진을 올리듯, GitHub에 내 코드를 올려서 보관하고 공유
• 전 세계 개발자들이 자기 프로젝트를 GitHub에 올려요
• 내 컴퓨터가 고장 나도 코드가 안전하게 보관됨
• 무료!
📌 Repository(저장소)가 뭔가요?
Repository(레포지토리, 줄여서 "레포")는 "프로젝트 폴더"예요.
우리의 etf-dashboard 폴더를 GitHub에 올리면, 그게 하나의 Repository!
비유: 인스타그램의 "게시물 하나" = GitHub의 "Repository 하나"

Step 2. GitHub 가입하기 (7분)

브라우저에서 아래 주소로 이동:

https://github.com

오른쪽 위 "Sign up" 버튼을 클릭하세요.

가입 화면이 나오면 차례대로:

1) "Enter your email"
이메일 주소를 입력하고 "Continue" 클릭

2) "Create a password"
비밀번호를 만드세요 (8자 이상, 숫자 포함) → "Continue" 클릭

3) "Enter a username"
사용자 이름을 만드세요 (영어, 숫자, - 만 사용 가능)
예시: my-first-code, student-kim, etf-learner

💡 이 이름이 내 GitHub 주소가 돼요! (github.com/내이름)

"Continue" 클릭

4) "Would you like to receive product updates?"
이메일 수신 여부 - "n"을 입력해도 괜찮아요 → "Continue" 클릭

5) 퍼즐 인증이 나오면 화면의 퍼즐을 풀어주세요

6) "Create account" 클릭

7) 이메일 확인
이메일함에서 GitHub가 보낸 메일을 열어 인증 코드(6자리 숫자)를 확인하세요.
그 숫자를 화면에 입력하면 가입 완료!

🎉 가입 완료! 이제 GitHub 회원이에요!

Step 3. 새 Repository(저장소) 만들기 (5분)

GitHub에 로그인한 상태에서:

1) 오른쪽 위의 "+" 버튼 클릭
2) "New repository" 클릭

새 저장소 만들기 화면:

"Repository name" (저장소 이름):

etf-youtube-dashboard

"Description" (설명, 선택사항):

ETF YouTube Data Dashboard - My First Project

"Public" 선택 (다른 사람도 볼 수 있게)
(비공개로 하고 싶다면 "Private" 선택)

나머지 옵션은 건드리지 마세요!

맨 아래 초록색 "Create repository" 버튼 클릭!

빈 저장소가 만들어졌어요! 화면에 명령어들이 보일 텐데, 우리가 직접 따라할 거예요.

Step 4. 내 코드를 GitHub에 올리기 (10분)

Git Bash를 열고 프로젝트 폴더로 이동:

cd ~/etf-dashboard

아래 명령어를 한 줄씩 차례로 입력하세요:

1) Git 저장소 시작하기:

git init
📌 "이 폴더를 Git으로 관리할게!"라는 뜻

2) 내 이름과 이메일 등록하기:
(GitHub 가입할 때 쓴 이름과 이메일을 넣으세요)

git config user.name "내이름"
git config user.email "내이메일@gmail.com"
📌 Git에게 "이 코드를 만든 사람은 나야"라고 알려주는 것

3) .env 파일은 올리지 않도록 설정하기:

echo ".env" > .gitignore
⚠️ .env에는 API 키(비밀번호)가 있으니까 인터넷에 올리면 안 돼요!
.gitignore 파일은 "이 파일은 올리지 마"라고 Git에게 알려주는 목록

4) 모든 파일을 올릴 준비하기:

git add .
📌 "이 폴더의 모든 파일을 포장해줘"라는 뜻 (마지막에 점(.) 꼭 입력!)

5) 저장(커밋)하기:

git commit -m "My first ETF dashboard project"
📌 "이 상태를 기억해줘"라는 뜻. 따옴표 안의 메시지는 "이때 뭘 했는지" 메모하는 것

6) GitHub 저장소와 연결하기:

⚠️ "내깃허브이름"을 본인의 GitHub 사용자 이름으로 바꾸세요!
git remote add origin https://github.com/내깃허브이름/etf-youtube-dashboard.git

7) GitHub에 올리기!:

git branch -M main
git push -u origin main

GitHub 로그인 창이 나타날 수 있어요:

• 브라우저가 열리면 "Authorize" 또는 "허용" 클릭
• 또는 GitHub 사용자 이름과 비밀번호 입력

성공하면 이런 메시지가 나와요:

To https://github.com/내깃허브이름/etf-youtube-dashboard.git
 * [new branch]      main -> main

Step 5. GitHub에서 내 프로젝트 확인하기! (2분)

브라우저에서 아래 주소로 이동하세요 (내깃허브이름을 본인 이름으로):

https://github.com/내깃허브이름/etf-youtube-dashboard

내가 만든 파일들이 GitHub 웹사이트에 예쁘게 보여요!

이 주소를 친구에게 공유하면, 친구도 내 프로젝트를 볼 수 있습니다!

💡 .env 파일은 올라가지 않은 것을 확인하세요!
(비밀번호가 인터넷에 노출되면 안 되니까요)

✅ 체크포인트

  • GitHub에 가입했나요?
  • etf-youtube-dashboard 저장소를 만들었나요?
  • git push 후 GitHub 웹사이트에서 파일이 보이나요?
  • .env 파일이 GitHub에 올라가지 않았나요? (올라가면 안 돼요!)

🚨 막혔을 때

증상원인해결법
git push에서 "Authentication failed"로그인 실패브라우저에서 GitHub 로그인 확인. git push 다시 실행하면 로그인 창이 뜰 수 있어요
"remote origin already exists"이미 연결됨git remote remove origin 먼저 실행 후 다시 git remote add...
"Repository not found"저장소 이름 불일치GitHub 웹에서 저장소 이름 확인 후 URL 정확히 입력
아무것도 안 올라가요add/commit 미실행git add . → git commit -m "메시지" → git push 순서대로 다시
"Author identity unknown" 에러Git에 이름/이메일 미등록git config user.name "이름"과 git config user.email "이메일" 실행
