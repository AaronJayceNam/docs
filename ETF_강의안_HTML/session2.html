Part A. 잠깐! 왜 Claude Code인가요? (10분)

Step 1. 웹 Claude vs Claude Code CLI

📌 혹시 이런 생각 해보셨나요?
"그냥 웹사이트(claude.ai)에서 쓰면 되는 거 아닌가? 왜 굳이 까만 창에서 써야 해?"

정말 좋은 질문이에요! 둘 다 같은 Claude AI인데, 할 수 있는 일이 완전히 달라요.
🌐 웹 Claude (claude.ai) = "전화 상담"

브라우저에서 대화창에 글을 입력하면 Claude가 답변해줘요.

• 질문하면 답변해줌 ✅
• 글 써달라고 하면 써줌 ✅
• 코드를 보여달라고 하면 화면에 보여줌 ✅

• 내 컴퓨터에 파일을 직접 만들어줌? ❌
• 프로그램을 직접 설치해줌? ❌
• 웹사이트를 열어서 조사해줌? ❌
• 내 프로젝트 폴더를 읽고 수정? ❌

💻 Claude Code (CLI) = "출장 전문가"

터미널에서 실행하면 Claude가 내 컴퓨터에 직접 와서 일해줘요!

• 질문하면 답변해줌 ✅
• 글 써달라고 하면 써줌 ✅
• 코드를 보여달라고 하면 화면에 보여줌 ✅

• 내 컴퓨터에 파일을 직접 만들어줌! ✅
• 프로그램을 직접 설치해줌! ✅
• 웹사이트를 열어서 조사해줌! ✅ (MCP 사용)
• 내 프로젝트 폴더를 읽고 수정! ✅

📌 한 줄 비유:
웹 Claude = 전화로 "이렇게 해보세요~" 라고 말만 해주는 상담사
Claude Code = 직접 우리 집에 와서 손으로 해주는 출장 전문가

Step 2. 왜 사람들이 Claude Code에 열광할까?

① "말만 하면 프로그램이 완성돼요"

웹 Claude: "이런 코드를 작성하세요" → 내가 복사해서 붙여넣고, 파일 만들고, 실행해야 함
Claude Code: "유튜브 대시보드 만들어줘" → 파일 생성부터 실행까지 다 해줌!

② "내 컴퓨터를 이해해요"

웹 Claude: 내 컴퓨터에 뭐가 설치됐는지, 어떤 파일이 있는지 전혀 모름
Claude Code: 내 폴더 구조, 설치된 프로그램, 파일 내용을 다 파악하고 맞춤 작업

③ "MCP로 초능력 추가 가능"

웹 Claude: 정해진 기능만 사용 가능
Claude Code: MCP를 장착하면 웹 브라우저 조종, 파일 관리, 데이터베이스 연결 등 무한 확장!

④ "대화가 아니라 '작업'을 시킬 수 있어요"

웹 Claude: "이렇게 하면 됩니다" (설명)
Claude Code: "다 만들었습니다. 실행해볼까요?" (실제 실행)

💡 실제 사례:
1회차에서 "유튜브 모니터 대시보드 만들어줘"라고 했을 때 기억나시죠?
웹 Claude였다면 코드만 보여주고 끝이었을 거예요.
Claude Code는 파일을 만들고, 필요한 패키지를 설치하고, 실행까지 해줬어요!

이것이 바로 전 세계 개발자들이 Claude Code에 열광하는 이유입니다.

Part B. 지난 시간 복습 (5분)

Step 3. 1회차에서 배운 것 돌아보기

지난 시간에 우리가 한 일을 기억해봅시다:

1) Windows Terminal 설치

컴퓨터와 대화하는 "까만 창"을 설치했어요.

2) Git & Git Bash 설치

코드 타임머신(Git)과 Claude가 잘 동작하는 터미널(Git Bash)을 설치했어요.

3) Node.js 설치

Claude Code를 돌리는 엔진을 설치했어요.

4) Claude Code 설치 & 첫 실행

npm install -g @anthropic-ai/claude-code로 AI 비서를 설치하고, 유튜브 대시보드를 만들어달라고 부탁했어요!

Step 4. 설치가 잘 되어 있는지 확인

Git Bash를 열고 아래 명령어를 하나씩 입력해보세요:

git --version
node --version
claude --version

세 가지 모두 숫자(버전)가 나오면 OK!

💡 만약 하나라도 안 되면 1회차 강의안을 다시 보면서 설치해주세요.

Part C. 명령 프롬프트(cmd)에서 Claude 실행하기 (10분)

Step 5. 명령 프롬프트(cmd)가 뭔가요?

📌 지난 시간에 터미널 종류를 배웠죠? 다시 정리하면:

• 명령 프롬프트(cmd) = 윈도우에 원래 있는 기본 터미널 (자전거 🚲)
• PowerShell = cmd의 업그레이드 버전 (전동 자전거 🛵)
• Git Bash = Git과 함께 온 특별한 터미널 (전동 킥보드 🛴)

지난 시간엔 Git Bash에서만 Claude를 실행했는데,
사실 명령 프롬프트(cmd)에서도 Claude를 실행할 수 있어요!

Step 6. cmd에서 Claude 실행해보기

윈도우 키를 누르고 아래를 검색하세요:

cmd

"명령 프롬프트"가 나오면 클릭! 까만 창이 열려요.

이제 Claude를 실행해봅시다:

claude

🎉 잘 실행되면 축하해요! 바로 Step 8로 넘어가세요.

⚠️ "'claude'은(는) 내부 또는 외부 명령...이 아닙니다" 라고 나왔나요?
걱정 마세요! 이건 컴퓨터가 Claude가 어디 있는지 모르는 거예요.
바로 다음 Step에서 길을 알려줄 거예요!

Step 7. Claude 경로(PATH) 설정하기

📌 경로(PATH)가 뭔가요?
비유: 친구 집에 놀러 가려면 "주소"를 알아야 하죠?
컴퓨터도 프로그램을 실행하려면 "그 프로그램이 어디에 있는지" 알아야 해요.
PATH는 "자주 가는 주소 목록"이에요. 여기에 Claude의 주소를 추가해주면 어디서든 claude를 실행할 수 있어요!

① 먼저 Claude가 어디 설치됐는지 찾기

Git Bash를 열고 아래 명령어를 입력하세요:

where claude

이런 결과가 나올 거예요 (사람마다 조금 다를 수 있어요):

C:\Users\여러분이름\AppData\Roaming\npm\claude
💡 이 경로를 마우스로 드래그해서 선택한 뒤 Ctrl+C로 복사해두세요!

② PATH에 추가하기 (그림으로 따라하기)

1단계: 윈도우 키를 누르고 아래를 검색하세요:

환경 변수

"시스템 환경 변수 편집"을 클릭하세요.

2단계: "시스템 속성" 창이 뜨면 아래쪽 "환경 변수(N)..." 버튼을 클릭하세요.

3단계: 위쪽 "사용자 변수" 목록에서 "Path"를 찾아 클릭하고 "편집(E)..." 버튼 클릭

4단계: "새로 만들기(N)" 버튼 클릭 후, 아까 복사한 경로에서 폴더 부분만 붙여넣기:

C:\Users\여러분이름\AppData\Roaming\npm
📌 주의! 끝에 \claude는 빼고, 폴더 경로만 넣으세요!
(예: C:\Users\홍길동\AppData\Roaming\npm\claude에서 \claude를 지운 것)

5단계: "확인" 버튼을 3번 눌러서 모든 창을 닫으세요.

③ 확인하기

⚠️ 중요: 열려 있던 cmd 창을 모두 닫고 새로 열어야 합니다!
(환경 변수는 새로 열리는 창부터 적용돼요)

cmd를 새로 열고:

claude --version

버전 숫자가 나오면 성공!

claude

🎉 이제 명령 프롬프트에서도 Claude Code가 실행됩니다!

✅ 중간 체크포인트

  • cmd에서 claude --version을 쳤을 때 숫자가 나왔나요?
  • cmd에서 claude를 쳤을 때 Claude Code가 실행되나요?

Part D. MCP가 뭔가요? (10분)

Step 8. MCP 쉽게 이해하기

📌 MCP = Model Context Protocol (모델 컨텍스트 프로토콜)

어려운 이름이지만 쉽게 설명하면:

Claude는 매우 똑똑한 AI이지만, 기본 상태에서는 할 수 없는 것들이 있어요:
• 웹사이트를 직접 열어서 보기 ❌
• 여러분의 구글 캘린더 확인하기 ❌
• 파일을 검색하거나 정리하기 ❌

MCP는 Claude에게 "초능력"을 추가해주는 장치예요!
🎮 비유: 게임 캐릭터의 아이템

게임에서 캐릭터가 맨몸이면 힘이 약하지만, 칼을 장착하면 공격이 세지고, 방패를 장착하면 방어가 되죠?

MCP도 똑같아요!

• Playwright MCP를 장착하면 → Claude가 웹 브라우저를 조종 가능! 🌐

• 파일 시스템 MCP를 장착하면 → Claude가 폴더/파일을 정리 가능! 📁

• GitHub MCP를 장착하면 → Claude가 GitHub을 사용 가능! 🐙

📌 한 줄 정리:
MCP = Claude에게 새로운 능력을 추가해주는 "플러그인" 같은 것!
필요한 MCP만 골라서 장착하면, Claude가 훨씬 더 많은 일을 할 수 있어요.

Step 9. 유용한 MCP 구경하기

세상에는 다양한 MCP가 있어요. 대표적인 것들을 구경해봅시다:

🌐 Playwright MCP

Claude가 웹 브라우저를 직접 열고, 클릭하고, 글을 읽을 수 있게 해줘요. "이 웹사이트에서 정보 좀 가져와줘"가 가능!

📁 Filesystem MCP

Claude가 컴퓨터의 파일과 폴더를 읽고, 만들고, 정리할 수 있게 해줘요. "바탕화면 파일 정리해줘"가 가능!

🐙 GitHub MCP

Claude가 GitHub(개발자들의 코드 저장소)에서 코드를 검색하고, 이슈를 만들 수 있게 해줘요.

📊 Brave Search MCP

Claude가 인터넷을 검색할 수 있게 해줘요. "최신 뉴스 찾아줘"가 가능!

📋 Google Drive MCP

Claude가 구글 드라이브의 문서를 읽고 편집할 수 있게 해줘요.

💡 이 중에서 오늘은 Playwright MCP를 직접 설치해볼 거예요!
웹 브라우저를 AI가 조종하는 모습, 정말 신기하답니다!

Part E. Playwright MCP 설치하고 첫 체험! (10분)

Step 10. Playwright MCP가 뭔가요?

📌 Playwright(플레이라이트)는 원래 웹 브라우저를 자동으로 조종하는 도구예요.

비유: 리모컨으로 TV를 조종하듯, Playwright는 "리모컨"으로 웹 브라우저를 조종해요.

Playwright MCP = 이 리모컨을 Claude에게 쥐어주는 것!
Claude가 리모컨을 받으면 직접 웹사이트를 열고, 버튼을 클릭하고, 글을 읽을 수 있어요.
Playwright MCP 없을 때

나: "네이버에서 오늘 날씨 알려줘"
Claude: "죄송하지만 저는 웹사이트에 접속할 수 없어요..." 😢

Playwright MCP 있을 때

나: "네이버에서 오늘 날씨 알려줘"
Claude: (브라우저를 열고 네이버에 접속해서 날씨를 확인) "서울 기온은 15도입니다!" 🎉

Step 11. Playwright MCP 설치하기 - 이것도 Claude에게 부탁!

📌 놀라운 점: MCP 설치도 Claude Code한테 부탁하면 돼요!
복잡한 명령어를 외울 필요가 없어요. 그냥 말로 시키면 됩니다.

Git Bash 또는 cmd를 열고 프로젝트 폴더로 이동하세요:

cd ~/etf-dashboard

Claude Code를 실행하세요:

claude

Claude의 > 프롬프트에서 이렇게 입력하세요:

Playwright MCP 설치해줘

🎉 이게 끝이에요! Claude가 알아서:

1. 필요한 패키지를 확인하고
2. 설치 명령어를 실행하고
3. MCP 서버를 등록해줍니다

💡 Claude가 "이 명령어를 실행해도 될까요?" 같은 질문을 하면 "y"를 입력하고 Enter!

이것이 바로 Claude Code의 힘이에요!
웹 Claude였다면 "이런 명령어를 입력하세요"라고 알려주기만 했을 거예요.
Claude Code는 직접 설치까지 해줍니다!

설치 확인하기

설치가 끝나면, 한 번 나갔다가 다시 들어올게요:

/exit

아래 명령어로 MCP가 잘 추가됐는지 확인:

claude mcp list

"playwright"가 목록에 보이면 성공!

Step 12. Playwright MCP 첫 체험!

다시 Claude를 실행하고:

claude

Claude의 > 프롬프트에서 아래를 입력해보세요:

네이버(naver.com)에 접속해서 메인 뉴스 헤드라인 5개를 알려줘
📌 무슨 일이 일어나는 거예요?
1. Claude가 Playwright MCP를 사용해서 브라우저를 엽니다
2. 네이버 웹사이트에 자동으로 접속합니다
3. 페이지의 내용을 읽어옵니다
4. 읽어온 정보를 정리해서 우리에게 알려줍니다

이 모든 것이 자동으로 일어나요!
💡 Claude가 "도구를 사용해도 될까요?" 같은 질문을 하면 "y"를 입력하고 Enter를 누르세요.

Step 13. 프롬프트 잘 쓰는 법

Playwright MCP에게 부탁할 때, 어떻게 말하느냐에 따라 결과가 달라져요!

🔴 이렇게 말하면 잘 안 돼요

"인터넷에서 뭐 좀 찾아줘" (너무 막연해요)

"ETF 정보 알려줘" (어디서 찾으라는 건지 모르겠어요)

🟢 이렇게 말하면 잘 돼요

"naver.com에 접속해서 '서울 날씨'를 검색하고 결과를 알려줘" (구체적!)

"youtube.com에서 'ETF 투자'를 검색하고 상위 5개 영상 제목을 표로 정리해줘" (사이트 + 행동 + 결과 형식!)

💡 프롬프트 잘 쓰는 공식:

"[어떤 사이트]에 접속해서 [무엇을] 찾고, [어떤 형식으로] 정리해줘"

1. 어떤 사이트에 갈지 알려주기 (예: naver.com, youtube.com)
2. 무엇을 찾을지 구체적으로 말하기 (예: "ETF 검색 결과 상위 5개")
3. 결과를 어떻게 정리할지 말하기 (예: "표로 정리해줘", "파일로 저장해줘")

Part F. 실전! Playwright로 ETF 시장 조사하기 (15분)

Step 14. ETF 시장 동향 파악하기

이제 진짜 업무에 쓸 수 있는 프롬프트를 배워봅시다! 하나씩 따라해보세요.

① ETF 인기 영상 트렌드 파악

유튜브(youtube.com)에 접속해서 "ETF 투자"를 검색하고,
상위 10개 영상의 제목, 채널명, 조회수, 업로드 날짜를
표로 정리해줘.

그리고 최근 어떤 ETF 주제가 인기인지
트렌드를 한 줄로 요약해줘.
📌 이 프롬프트의 포인트:
• "상위 10개"로 범위를 정해줌

Part G. 실전! 책에 대한 유튜브 반응 분석하기 (20분)

Step 16. 왜 유튜브 반응을 분석할까요?

📌 책을 출판하고 나면, 독자들이 어떻게 반응하는지 아는 것이 매우 중요해요.

특히 2쇄(두 번째 인쇄)를 준비할 때는:
• 독자들이 어떤 부분을 좋아했는지?
• 어떤 부분이 어렵다고 했는지?
• 추가해달라는 내용이 있는지?

이런 정보를 모아야 더 좋은 책을 만들 수 있어요.
유튜브에는 책 리뷰, 서평 영상이 많기 때문에 진짜 독자 반응을 확인하기 좋습니다!

Step 17. 유튜브에서 책 반응 검색하기

Claude에게 아래 프롬프트를 입력해보세요:

① 기본 검색: 저자명으로 영상 찾기

유튜브(youtube.com)에 접속해서 "배재규"를 검색하고,
관련 영상 10개의 제목, 채널명, 조회수, 업로드 날짜를
표로 정리해줘.

어떤 영상이 가장 인기 있는지,
영상들의 주요 주제가 무엇인지 분석해줘.
💡 저자 이름으로 검색하면 인터뷰, 서평, 강연 등 다양한 영상이 나와요!

② 책 제목으로 검색하기

유튜브(youtube.com)에서 "배재규 ETF"를 검색하고,
나오는 영상들을 분석해줘.

다음 내용을 정리해줘:
1. 영상 목록 (제목, 채널명, 조회수) - 표로
2. 영상들이 책의 어떤 내용을 주로 다루는지
3. 댓글이나 영상 설명에서 보이는 독자 반응 키워드
4. 전반적인 반응이 긍정적인지 부정적인지 평가

③ 서평/리뷰 영상 깊이 분석

유튜브에서 "배재규" 검색 결과 중
조회수가 가장 높은 영상 하나에 들어가서,
영상 설명(더보기)과 댓글 내용을 읽어줘.

다음을 분석해줘:
1. 영상에서 다루는 핵심 주제
2. 댓글에서 사람들이 가장 많이 언급하는 내용
3. 칭찬하는 부분 vs 아쉬워하는 부분
4. 독자들이 더 알고 싶어하는 주제
📌 댓글 분석이 강력한 이유:
책 리뷰 영상의 댓글에는 진짜 독자들의 솔직한 의견이 있어요.
"이 부분이 도움됐다", "이건 좀 어려웠다" 같은 피드백을
AI가 자동으로 분류해주니까 엄청 편합니다!

Step 18. 온라인 서점에서 독자 리뷰 분석하기

① 네이버 책 리뷰 모아보기

네이버(naver.com)에서 "배재규" 책을 검색하고,
네이버 책(book) 탭에서 나오는 책 정보와
독자 리뷰를 최대한 많이 읽어서 정리해줘.

다음을 분석해줘:
1. 평점 분포 (별점이 어떻게 분포하는지)
2. 긍정 리뷰에서 반복되는 키워드 Top 5
3. 부정 리뷰에서 반복되는 키워드 Top 5
4. 독자들이 가장 유용했다고 꼽는 챕터나 내용
5. "이런 내용도 있었으면 좋겠다"는 요청 사항

② 알라딘/YES24 리뷰 비교

알라딘(aladin.co.kr)에 접속해서 "배재규"를 검색하고,
책의 독자 리뷰를 읽어줘.

그 다음 YES24(yes24.com)에서도 같은 책을 검색해서
리뷰를 읽어줘.

두 사이트의 리뷰를 비교해서:
1. 공통적으로 나오는 칭찬 포인트
2. 공통적으로 나오는 개선 요청
3. 사이트별로 다른 반응이 있다면 정리
이렇게 비교 분석해줘.
💡 여러 사이트를 비교하는 이유:
유튜브, 네이버, 알라딘, YES24 — 각 플랫폼의 독자층이 달라요.
여러 곳의 반응을 종합하면 더 정확한 그림이 나옵니다!

Step 19. 2쇄를 위한 개선 방향 잡기

2쇄(두 번째 인쇄)를 어떻게 개선할지 Claude에게 분석을 부탁합니다.

① 반응 종합 분석 요청

지금까지 유튜브, 네이버, 서점에서 수집한
배재규 저자의 책에 대한 반응을 종합해서 분석해줘.

다음 형식으로 정리해줘:

[독자가 좋아한 점 Top 5]
- 구체적인 근거(어떤 댓글/리뷰에서 나왔는지)와 함께

[독자가 어려워한 점 Top 5]
- 어떤 내용이 어렵다고 했는지, 왜 어렵다고 느꼈는지

[독자가 추가를 원하는 내용 Top 5]
- 어떤 주제/챕터를 원하는지

[2쇄 개선 제안]
- 위 분석을 바탕으로 구체적인 개선 방안 5가지

② 경쟁 도서와 비교 분석

네이버 책(book.naver.com)에서 "ETF 투자" 관련 베스트셀러를
검색하고, 상위 5권의 책 정보와 평점을 정리해줘.

그리고 배재규 저자의 책과 비교해서:
1. 우리 책만의 강점 (다른 책에 없는 것)
2. 경쟁 도서에서 배울 점
3. 2쇄에서 차별화할 수 있는 포인트
를 분석해줘.

③ 독자 눈높이에 맞는 내용 편집 방향

유튜브에서 "ETF 초보", "ETF 입문", "ETF 쉽게"를
각각 검색해서 인기 영상들의 제목과 내용을 분석해줘.

초보 투자자들이 가장 궁금해하는 것이 무엇인지 파악하고,
2쇄에서 다음을 어떻게 개선하면 좋을지 제안해줘:

1. 초보자가 이해하기 쉽게 바꿔야 할 설명이나 용어
2. 추가하면 좋을 그림, 도표, 예시
3. 새로 넣으면 좋을 Q&A 또는 FAQ 주제
4. 챕터 순서나 구성 변경 제안
📌 이 분석이 대단한 이유:
보통 이런 시장 조사를 하려면 설문조사를 하거나 리서치 회사에 맡겨야 해요.
그런데 Claude + Playwright MCP를 쓰면 지금 바로, 무료로 할 수 있어요!
실제 독자 반응을 기반으로 하니까 설문조사보다 더 솔직한 의견을 모을 수 있답니다.
