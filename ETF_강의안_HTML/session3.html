Part A. GitHub이 무엇인가요? (15분)

Step 1. GitHub 쉽게 이해하기

🎓 알기 쉽게 설명!

여러분이 그림을 그린다고 상상해보세요.

• 그림을 그리고 있는데, "아까! 이거 지우기 전으로 돌아가고 싶다!" → Ctrl+Z (되돌리기)
• 친구에게 그림을 보여주고 싶다! → 카카오톡으로 전송
• 친구와 같이 그림을 그리고 싶다! → 구글 도큐먼트로 공유

Git과 GitHub은 이 세 가지를 다 할 수 있는 "코드 버전"이에요!
💻 Git = 타임머신

내 컴퓨터에서 돌아가는 프로그램이에요.

• 파일을 수정할 때마다 "사진을 찍!" 하고 기록해둘요 (= commit)
• 언제든지 과거로 돌아갈 수 있어요
• 1회차에서 설치한 그 Git이 바로 이거에요!

🌐 GitHub = 클라우드 저장소

Git으로 기록한 내용을 인터넷에 올려두는 곳이에요.

• 내 컴퓨터가 고장 나도 코드가 안전해요!
• 주소만 알려주면 누구나 볼 수 있어요
• 친구와 같이 작업할 수도 있어요

📌 한 줄 정리:
Git = 내 컴퓨터에서 파일의 역사를 기록하는 타임머신
GitHub = 그 기록을 인터넷에 백업하고 공유하는 웹사이트
(Git 없이 GitHub을 쓰 수 없고, GitHub 없이도 Git은 쓰 수 있어요!)

Step 2. 왜 GitHub이 중요한가요?

① 코드가 안전해요

비유: USB에 숙제만 저장하면... USB를 잃어버리면 끝! 😭
GitHub에 올려두면 컴퓨터가 고장 나도 안전해요. 👏

② 시간 여행이 가능해요

비유: 게임에서 "세이브 포인트"를 만들어두는 것처럼!
어제 만든 버전, 그저게 만든 버전... 언제든 돌아갈 수 있어요.

③ 다른 사람과 협업할 수 있어요

비유: 구글 독스처럼 여러 사람이 같은 파일을 편집할 수 있어요.
누가 어디를 고쳤는지 다 기록이 남아요!

④ 포트폴리오(이력서)가 돌요

"나는 이런 프로젝트를 만들었어요!"라고 보여줄 수 있는 곳!
개발자들은 취업할 때 GitHub 주소를 같이 내요.

Part B. Git 핵심 개념: Commit, Push, Pull (15분)

Step 3. Commit(커밃) = "사진 찍!"

📷 비유: 사진 찍기

여행을 가서 멋진 경치가 나오면 사진을 찍죄? 🎥

코드도 마찬가지에요!
파일을 수정하고, "이 상태를 기억해둘!" 하고 찍! 하는 것이 바로 commit(커밃)이에요.

사진에 "에펄탑에서" 같은 설명을 붙이듯, commit에도 메시지를 붙여요:
"첫 번째 대시보드 완성" 처럼요!
📝 파일을 수정한다
↓
📦 git add . (사진 찍을 준비! "이 파일들 찍어줄게")
↓
📷 git commit -m "메시지" (찍! 기억해둘!)
# 예시: 파일을 수정한 후
git add .                              # "이 파일들 기억해줄"
git commit -m "대시보드 색상 변경"    # "찍!" + 설명 메모
💡 commit 메시지 잘 쓰는 법:
❌ 나쁜 예: "수정함", "업데이트" (무엇을 수정했는지 모르겠죄!)
✅ 좋은 예: "그래프 색상을 파란색으로 변경", "검색 기능 추가"

Step 4. Push(푸시) = "인터넷에 올리기!"

📤 비유: 사진을 클라우드에 업로드

휴대폰으로 사진을 찍으면, 휴대폰에만 있죄?
그 사진을 구글 포토나 아이클라우드에 올려두면 어디서든 볼 수 있고 안전하죄!

push = 내 컴퓨터에 있는 commit(사진)을 GitHub(클라우드)에 올리는 것이에요!
💻 내 컴퓨터에서 commit 완료
↓
📤 git push (GitHub에 올리기!)
↓
🌐 GitHub 웹사이트에서 누구나 볼 수 있음!
# commit을 한 후에
git push origin main    # "GitHub에 올려!"
📌 origin이 무엇인가요?
origin = "GitHub 저장소의 별명"이에요. "원래 저장소"라는 뜻이에요.
main = "메인 브랜치"라는 뜻인데, 지금은 "기본 버전"이라고만 알면 둘요!

Step 5. Pull(풀) = "인터넷에서 내려받기!"

📥 비유: 클라우드에서 사진 내려받기

회사에서 파일을 수정해서 GitHub에 올려둴다고 해볼게요.
집에 와서 그 수정된 내용을 받아오고 싶으면?

pull = GitHub에 있는 최신 내용을 내 컴퓨터로 내려받는 것이에요!
🌐 GitHub에 새로운 내용이 있음
↓
📥 git pull origin main (내려받기!)
↓
💻 내 컴퓨터가 최신 상태로 업데이트!
# 다른 컴퓨터에서, 또는 GitHub에서 직접 수정한 후
git pull origin main    # "최신 내용 내려받기!"

Step 6. 전체 흐름 한눈에 보기

📌 일상생활 비유로 정리:

1. 파일 수정 = 그림을 그린다
2. git add = "이 그림 찍을 거야" (카메라 준비)
3. git commit = "찍!" (사진 찍기)
4. git push = "클라우드에 업로드" (안전하게 보관)
5. git pull = "클라우드에서 다운로드" (최신 버전 받기)
📋 요약 표
명령어쉬운 설명하는 일
git add ."이거 찍어줄"변경된 파일을 준비
git commit -m "...""찍! 기억해!"현재 상태를 저장
git push"인터넷에 올려!"GitHub에 업로드
git pull"최신거 내려받아!"GitHub에서 다운로드
git status"지금 상태가 어때?"변경 사항 확인
git log"기록 보여줄"commit 역사 확인

✅ 중간 체크포인트

  • Git과 GitHub의 차이를 설명할 수 있나요?
  • commit이 "사진 찍기"라는 것을 이해했나요?
  • push는 "올리기", pull은 "내려받기"라는 것을 이해했나요?

1) Git 저장소 시작 + 첫 번째 commit:

이 폴더를 Git 저장소로 초기화해 줌.

그리고 모든 파일을 "My first ETF dashboard project"라는 메시지로 commit해 줌.

   - 조회수 TOP 10 영상 막대 그래프 (Chart.js 사용)
   - 채널별 영상 수 파이 차트
   - 전체 영상 목록 테이블 (제목 클릭하면 유튜브로 이동)
4. 디자인: 까끔한 다크 테마, 카드형 레이아웃
5. JSON 데이터를 HTML 파일 안에 직접 포함시겠줌
6. 모바일에서도 보기 좋게 반응형으로 만들어줌

Enter! Claude가 대시보드를 만들기 시작합니다.
